## Templates To Create Yourself

For weeks without pre-made templates, use these structures:

### Prompt Engineering Playbook Structure:
```markdown
# Prompt Engineering Playbook

## Effective Patterns
1. [Pattern name]: [Description, example, when to use]

## Best Practices
- [Practice 1]
- [Practice 2]

## What Works
- [Insight 1]
- [Insight 2]

## What Doesn't Work
- [Anti-pattern 1]
- [Anti-pattern 2]

## Prompt Templates
### Template 1: [Name]
```
[Actual prompt template with placeholders]
```

## Parameter Settings
- Temperature: [When to use what]
- Max tokens: [Recommendations]
```

### UX Flow Template:
```markdown
# UX Flow: [Name]

**Scenario:** [Description]

**Steps:**
1. User: [Action/Input]
   AI: [Response]
   UI State: [What user sees]

2. User: [Action/Input]
   AI: [Response]
   UI State: [What user sees]

[Continue...]

**Success Criteria:** [How we know it worked]

**Error Handling:** [What if it fails]

**Screenshots:** [Add if available]
```

### GTM Strategy Template:
```markdown
# Go-to-Market Strategy

## 1. Market Analysis
- TAM/SAM/SOM
- Market trends
- Regulatory considerations

## 2. Competitive Landscape
- Direct competitors
- Indirect competitors
- Competitive advantages
- Differentiation

## 3. Positioning
- Value proposition
- Target segments
- Messaging framework

## 4. Business Model & Pricing
- Revenue model
- Pricing strategy
- Unit economics

## 5. Launch Strategy
- Phase 1: Beta
- Phase 2: Public launch
- Phase 3: Scale

## 6. Growth Strategy
- Acquisition channels
- Activation tactics
- Retention mechanisms

## 7. Risks & Mitigations
```

## Pro Tips

**Organization:**
- Create a folder for each week
- Name files clearly (e.g., `week-3-metrics-dashboard-my-product.md`)
- Keep raw templates separate from your filled versions

**Version Control:**
- Use Git to track your progress
- Commit after completing each deliverable
- Your commit history shows your journey

**Portfolio Presentation:**
- Create a master README linking all deliverables
- Polish the most important docs (Weeks 9-11)
- Demo video and presentation deck are most important for interviews
