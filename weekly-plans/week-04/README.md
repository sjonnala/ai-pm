## üéì Key Concepts to Master

### Roadmapping:
- **Theme-Based Roadmaps:** Organized by strategic initiatives, not features
- **Now-Next-Later:** Communicating uncertainty in timelines
- **OKRs to Roadmap:** Connecting objectives to execution
- **Stakeholder Communication:** Different views for different audiences

### Prioritization:
- **RICE Framework:** Quantitative scoring for features
- **Value vs. Effort Matrix:** Visual prioritization
- **Strategic Alignment:** Features must tie to strategy
- **Opportunity Cost:** Choosing means saying no

### AI Transformation:
- **Crawl-Walk-Run:** Phased approach to AI adoption
- **Automation ‚Üí Augmentation ‚Üí Agentic:** Evolution of AI capabilities
- **Change Management:** Bringing teams along the journey
- **Technical Debt:** Balancing innovation with stability

## üí° Interview Questions You Should Be Able to Answer

1. "How do you build a product roadmap?"
   - **Framework:** Strategy ‚Üí Themes ‚Üí Initiatives ‚Üí Features + Prioritization + Metrics

2. "How do you prioritize features?"
   - **Answer:** RICE framework + strategic alignment + stakeholder input

3. "How would you add AI to [existing product]?"
   - **Framework:** Crawl-Walk-Run + Use transformation playbook approach

4. "You have limited engineering resources. How do you decide what to build?"
   - **Answer:** Impact vs. Feasibility + RICE + Strategic bets vs. Quick wins

5. "How do you communicate roadmap to executives?"
   - **Answer:** Themes, outcomes, metrics - not feature lists

6. "Your roadmap has changed 3 times this quarter. How do you handle that?"
   - **Answer:** Embrace change, communicate why, maintain themes even if features shift
