### **Day 1 (Monday): Write AI PRD - Part 1**

**Morning Session (2-3 hours)**
- **Topic:** PRD Core Sections
- **Write:**

  **1. Executive Summary**
  - Problem statement
  - Solution overview
  - Success metrics
  - Target launch timeline

  **2. Background & Context**
  - Why this problem matters
  - Market opportunity
  - User research insights
  - Competitive landscape

  **3. Goals & Success Metrics**
  - North Star Metric
  - Product metrics (AARRR)
  - ML metrics
  - Business metrics
  - Guardrail metrics

  **4. User Stories & Use Cases**
  - Primary user personas (2-3)
  - 10+ user stories
  - 5+ detailed use cases with flows
  - Jobs to be done

**Evening Session (2-3 hours)**
- **Topic:** Functional Requirements
- **Write:**

  **5. Functional Requirements**
  - Core features (MVP)
  - User-facing capabilities
  - AI behaviors
  - User controls and overrides
  - Feedback mechanisms

  **6. User Experience**
  - Key user flows
  - Interaction patterns
  - UI/UX principles
  - Multimodal considerations

**Deliverable:** PRD sections 1-6 complete

### **Day 2 (Tuesday): Write AI PRD - Part 2 (AI-Specific)**

**Morning Session (2-3 hours)**
- **Topic:** AI-Specific Requirements
- **Write:**

  **7. Data Requirements**
  - Data sources (training + inference)
  - Data volume and quality needs
  - Labeling requirements
  - Data pipeline
  - Privacy and compliance
  - Data refresh strategy

  **8. Model Requirements**
  - Model type/architecture
  - Pre-trained vs. fine-tuned
  - Input/output specifications
  - Performance thresholds
  - Context window (for LLMs)
  - Model hosting (API vs. self-hosted)
  - Cost modeling

**Evening Session (2-3 hours)**
- **Topic:** Evaluation, Safety & Technical Specs
- **Write:**

  **9. Evaluation Criteria**
  - Offline evaluation metrics
  - Online evaluation plan
  - Human evaluation rubric
  - A/B test design
  - Success thresholds

  **10. Safety & Guardrails**
  - Content filtering
  - Rate limiting
  - Error handling
  - Fallback behaviors
  - Human-in-the-loop triggers
  - Monitoring and alerting

  **11. Technical Requirements**
  - Non-functional requirements (latency, availability, etc.)
  - System architecture (high-level)
  - API specifications
  - Integration points
  - Security requirements

  **12. Edge Cases & Failure Modes**
  - 20+ edge cases
  - Expected behaviors
  - Mitigation strategies

**Deliverable:** Complete AI PRD (all sections)

### **Day 3 (Wednesday): Build Prototype - Part 1**

**Morning Session (2-3 hours)**
- **Topic:** Core Prototype Development
- **Build:**
  - Set up prompting environment
  - Create system prompts
  - Implement main user flow
  - Test with initial scenarios

**Approach:**
  - Use ChatGPT, Claude, or similar
  - Document all prompts used
  - Create reusable prompt templates
  - Build conversation flows
  - Handle state/context

**Tools:**
  - ChatGPT Plus (for Custom GPTs)
  - Claude (for Projects)
  - OR use API with simple UI
  - OR use no-code tools (Voiceflow, Landbot, etc.)

**Evening Session (2-3 hours)**
- **Topic:** Refine & Expand
- **Activity:**
  - Add error handling
  - Implement guardrails (within prompts)
  - Test edge cases from PRD
  - Refine prompts for quality
  - Add personality/tone
  - Test with 5-10 scenarios

**Document:**
  - Prompt engineering decisions
  - What works vs. doesn't
  - Limitations discovered
  - Workarounds implemented

---

### **Day 4 (Thursday): Build Prototype - Part 2 + User Testing**

**Morning Session (2-3 hours)**
- **Topic:** Advanced Features & Polish
- **Build:**
  - Multi-turn conversation handling
  - Context management
  - Output formatting
  - Citations/sources (if applicable)
  - Any multimodal features
  - Polish the experience

**Evening Session (2-3 hours)**
- **Topic:** User Testing
- **Activity:**
  - Recruit 5-7 test users
  - Give them scenarios (don't tell them what to say)
  - Observe interactions
  - Document:
    - What works
    - What confuses users
    - Where AI fails
    - Unexpected uses
    - Feedback and suggestions
  - Iterate based on findings
  - Refine prompts and flows

**Deliverable:** Working, tested prototype

### **Day 5 (Friday): Design UX Flows + Architecture**

**Morning Session (2-3 hours)**
- **Topic:** Create 5+ Detailed UX Flows
- **Document flows for:**
  1. **Happy Path:** User gets perfect result
  2. **Ambiguous Input:** User is unclear, AI clarifies
  3. **Multi-Turn:** Complex task requiring back-and-forth
  4. **Error State:** AI fails or can't help
  5. **Edge Case:** Unusual but valid scenario
  6. **(Bonus) Power User:** Advanced features
  7. **(Bonus) First-Time User:** Onboarding

**For each flow:**
  - Step-by-step user journey
  - Exact inputs (screenshots if possible)
  - AI responses
  - UI states
  - Error handling
  - Success criteria

**Tools:**
  - Whimsical, Figma, Miro
  - Or simple markdown with screenshots

**Evening Session (2-3 hours)**
- **Topic:** Technical Architecture
- **Create:**
  - System architecture diagram
  - Data flow diagram
  - Component breakdown:
    - User interface
    - API/backend
    - LLM integration
    - Data sources
    - Vector database (if RAG)
    - Monitoring/logging
  - Technology stack
  - Deployment considerations

**Deliverable:** UX flows + architecture diagram

## üí° PRD Best Practices Reminder

**Make it:**
- **Clear:** Engineers can build from it
- **Comprehensive:** Covers all AI-specific needs
- **Crisp:** Concise, well-organized
- **Visual:** Diagrams, flows, mockups
- **Measurable:** Clear success metrics
- **Realistic:** Acknowledges limitations and trade-offs

**AI-Specific Sections Must Include:**
- Exact data requirements
- Model selection rationale
- Evaluation approach
- Failure mode handling
- Cost estimates
- Monitoring plan

## üéØ Demo Preparation Tips

**5-Minute Demo Structure:**
1. **Problem** (30 sec)
   - "Users struggle with X because Y"
2. **Solution** (30 sec)
   - "I built [product], which uses AI to Z"
3. **Demo** (3 min)
   - Show 2-3 key scenarios
   - Highlight AI capabilities
   - Show error handling
4. **Impact** (30 sec)
   - Success metrics
   - What you learned
5. **Q&A** (30 sec)
   - Anticipate questions

**Practice until it's smooth!**
